# 调试与运行（UTS 插件）

> 同步说明（来源：官方文档）
> - 参考页面：UTS 插件介绍 - 真机运行/自定义基座/断点调试
> - 同步时间：2025-09-13

## 真机运行

- 在 HBuilderX 中选择目标平台并运行到真机；
- 初次运行会进行依赖准备（如 iOS 的 `pod install`）。

提示：Android 需开启开发者选项与 USB 调试；iOS 需信任证书并配置签名。

## 自定义基座

- 当插件依赖较多原生能力时，建议制作自定义基座以加快调试迭代；
- 修改插件后重新安装/更新自定义基座，减少云打包等待时间。

## 断点调试

- Android：
  - 使用 Android Studio 附加到运行中的进程；
  - 在 UTS 转 Kotlin/Java 处与混编 Kotlin 文件中均可设置断点。
- iOS：
  - 使用 Xcode 连接运行中的进程；
  - 在 UTS 转 Swift/ObjC 处与混编 Swift 文件中设置断点。

## 日志与排查

- Android：`Logcat`；iOS：`Console`；
- 插件内适度增加日志与错误边界（try/catch），避免吞错。

## 常见陷阱

- 线程：UI 更新必须在主线程；
- 权限：记得动态申请并处理“不要再询问”；
- 包名/签名：与宿主工程一致，避免找不到类/方法或签名冲突；
- 依赖版本：Kotlin 2.2.0（HX 4.81 起）；Pod/Gradle 版本差异导致的 API 变化需同步适配。
